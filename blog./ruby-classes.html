Introduction to Ruby Classes

Classes in Ruby are like a template or blueprint for defining data as objects and the methods that can operate on that data. The idea of the class is to group things together into one unit so we can have access to them. Every time we create something it becomes an Object(a piece of data that has been created from the Class blueprint). Class defines a "state" and the "behaviour" of an object . They can be instantiated or having objects created that are instances of the class, we do that the following way:

obj = Object.new

When we use the dot on the class we send message to the class, in this case we have the "constructor" method whose purpose is to return us a new instance of the class. We define a class with the "class" keyword, and we name them with constants that begin with a capital letter. Lets create a Triangle class , initialize an object and give it a properties like side and lenght:

class Triangle
def initialize(side, length)
@side = side
@length = length
end
end

First what is the "initialize" method doing? The "initialize" method defines the creation of the object, or "initialize" is called whenever a new object is created with the class. We take its parametars and we turn them into instance variables "@side" and "@length" that we set them equal to the Object's side and the length. Instance variables store the side and the length of the Triangle. Instance variables are like any other other variable except they are visible to the Object they belong to, and can be passed between any other methods that are defined in that class.

Let's create two more methods where we will get the length of the Triangle and set a length ot the same and call those methods like this:

class Triangle
def initialize(side, length)
@side = side
@length = length
end

def length=(new_length)
@length = new_length
end

def length
@length
end
end

triangle = Triangle.new(3, 5)
triangle.length
>> 5
triangle.length =(10)
>> 10
triangle.length
>> 10

Now why do we use getter and setter methods? When we need to set , add or change an Object's state at some point we use setter method, or lets put it this way, "getters" and "setters" are methods that are responsible for setting an instance variable (setter), and retrieving the value of an instance variable (getter). Another interesting thing is that Ruby has built in methods to make the code cleaner and simpler that are equivalent to these getter and setter methods, we call them them the "attr_*" method family. Lets refactor the code above :

class Triangle
attr_reader :side , :length
attr_writer :length
def initialize(side, length)
@side = side
@length = length
end
end

Beautiful right! The "attr_reader" is an attribute of an object that allows its value to be viewed and called outside of the class, it is immutable and equivalent to the "getter" method. The "attr_writer" is an attribute of an object that allows its value to be changed, added etc., from outside of the class and equivalent to the "setter" method. These attributes are wrapped around our instance variables , and the instance variable that has the ability to get and set values can be also listed with the "attr_accessor" instead. This "attr_accessor" has the ability of both combined "attr_reader" and "attr_writer", it creates the reader and writer methods, and it gives us even better , simpler one-line code. So it would look something like this:

class Triangle
attr_reader :side
attr_accessor :length
def initialize(side, length)
@side = side
@length = length
end
end
