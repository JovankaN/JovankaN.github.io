Arrays

Arrays are an ordered collection of objects , its job is to store other objects .Any objects can be stored in an array including other arrays,hashes , classes, any objects at all.The contents of an array always remains the same in the order it was stored unless we want to move the objects around.Since there are more then one way to create a new array, some of them are created the following way:
Creating with the "Array.new" method :
color = Array.new
color.push "red"
puts color
=> red ;
Creating the literal way with the square brackets :
color = ["red", "green", "yellow"]
return color[0] => "red",
return color [1]=>
"green", return color [2]=>"yellow" ;
One thing that i want to point to in the second example is that in arrays each element has an index number, starting from 0 and on. We can access them by calling this number like in : color = ["red", "green", "yellow"] return color[0] => "red",;
When we called the array we reffered to the first elemenet in the array so we said return the first element in that array , and of course the counting starts from 0 in arrays and it increases by 1 like if we would like to call the second element we would call it like this: return color [1]=> "green"; and so on and so forth.

Hashes

Like the arrays, Hashes are collection of objects. A collection that contains keys and value pairs, where any key and any value can be any Ruby object.Hashes perform based on their keys.We access the value of a hash by refereing to the key element. There are different ways to create a hash, but one of them is by using the literal constructor the curly braces" {} " which looks like this:,
hash ={
color: "red"
tequila:"Mezcal"
}
return hash [:color] => "red" ;
Or we could print all of the values by simply writing:
hash ={
color: "red"
tequila:"Mezcal"
}
return hash.values =>["red", "Mezcal"] ;
I hope this blog helps you to get a better idea of Arrays and Hashes , their creation and the way we access their elements.
